Figma → Chakra UI

Гайд по правильному переносу дизайна: от токенов в Figma до нативных компонентов в Next.js без лишних пропсов.

Промпт

Контекст: Разработка фронтенда на Next.js + Chakra UI. Дизайнер только начинает делать макет. Нужен уникальный дизайн на базе компонентов Chakra UI.

Задача 1: Что дизайнер должен добавить в макет, чтобы корректно перенести UI Kit в тему Chakra UI?

Задача 2: Как правильно переносить дизайн из Figma в Chakra, чтобы в итоге использовать стандартные компоненты без кастомных пропсов?

Пример: <Button variant="solid" size="sm">Кнопка</Button> — выглядит как в макете автоматически.

ЧАСТЬ 1 Что дизайнер должен добавить в макет

1. Design Tokens — основа всего

Дизайнер должен создать в Figma Local Variables и Styles со строгой структурой, которая маппится 1-в-1 на Chakra тему.

Цвета — Color Variables

Figma Local Variables → colors naming 
brand/50, brand/100, brand/200 ... brand/900   ← primary palette
gray/50 ... gray/900
red/50 ... red/900                              ← для error states
green/50 ... green/900                          ← для success
Именно нотация название/числовое значение — она 1-в-1 ляжет в Chakra colors

Типографика — Text Styles

heading/4xl, heading/3xl, heading/2xl, heading/xl, heading/lg
body/lg, body/md, body/sm
label/lg, label/md, label/sm

Spacing — Numeric Variables (шаг 4px)

space/1 = 4px
space/2 = 8px
space/3 = 12px
space/4 = 16px   ← и далее

Радиусы, тени, border-width — тоже переменными.

2. Компонентные состояния

Для каждого компонента дизайнер обязан нарисовать все состояния и все размеры.

КомпонентОбязательные состояния
Buttondefault, hover, active, disabled, loading
Inputdefault, focus, error, disabled, filled
Carddefault, hover, selected
Badgeкаждый variant

И каждый size variant: sm, md, lg

3. Component Properties в Figma

Компоненты создаются через Figma Component Properties — прямой аналог пропсов Chakra:

Button
  ├── variant: solid | outline | ghost | link
  ├── colorScheme: brand | gray | red | green
  ├── size: sm | md | lg
  └── state: default | hover | disabled | loading

import { defineStyleConfig } from '@chakra-ui/react'

export const Button = defineStyleConfig({
  baseStyle: {
    fontWeight: '600',
    borderRadius: '8px',
    _focus: {
      boxShadow: '0 0 0 3px var(--chakra-colors-brand-200)',
    },
  },
  sizes: {
    sm: { h: '36px', px: '16px', fontSize: 'sm' },
    md: { h: '44px', px: '24px', fontSize: 'md' },
    lg: { h: '52px', px: '32px', fontSize: 'lg' },
  },
  variants: {
    solid: {
      bg: 'brand.600',
      color: 'white',
      _hover: { bg: 'brand.700', _disabled: { bg: 'brand.600' } },
      _active: { bg: 'brand.800' },
      _disabled: { opacity: 0.4, cursor: 'not-allowed' },
    },
    outline: {
      border: '1.5px solid',
      borderColor: 'brand.600',
      color: 'brand.600',
      _hover: { bg: 'brand.50' },
      _active: { bg: 'brand.100' },
    },
    ghost: {
      color: 'brand.600',
      _hover: { bg: 'brand.50' },
    },
  },
  defaultProps: {
    variant: 'solid',
    size: 'md',
    colorScheme: 'brand',
  },

ЧЕКЛИСТ Для дизайнера

  • ✓Color palette (50–900) для каждого цвета бренда
  • ✓Text Styles с именами heading/xl, body/md и т.д.
  • ✓Spacing variables кратные 4px
  • ✓Border radius variables (sm / md / lg)
  • ✓Shadow styles (sm / md / lg / xl)
  • ✓Каждый компонент — все variants × все sizes × все states
  • ✓Component Properties = variant, size, state, colorScheme
  • ✓Dark mode версии (если нужен темный режим)
  • ✓Breakpoints для адаптива: 480 / 768 / 992 / 1280 / 1536px
Главное правило: дизайнер работает в системе токенов → разработчик переносит токены в тему → компоненты используются нативно без лишних пропсов.
